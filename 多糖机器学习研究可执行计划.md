# 多糖机器学习研究可执行计划

## 1. 筛选结论

基于报告内容、当前仓库现状（只有报告、暂无数据与代码）、以及 `research-ideation` 的问题筛选标准，建议把方向收缩为：

| 方向 | 是否立刻启动 | 研究价值 | 当前可执行性 | 结论 |
| :--- | :--- | :--- | :--- | :--- |
| A. 多糖结构-功能多任务预测 | 是 | 高 | 高 | 作为主线 |
| D. 多源融合多糖知识图谱 | 是，但缩小范围 | 高 | 中高 | 作为数据底座 |
| F. 多糖/糖链自监督预训练 | 可以预研 | 中高 | 中 | 作为第二阶段增强 |
| B. 多模态谱图->结构 | 暂缓 | 很高 | 低 | 等数据对照集稳定后再做 |
| C. 约束生成式设计新多糖 | 暂缓 | 很高 | 低 | 依赖 A/D 先成熟 |
| E. 主动学习/实验闭环 | 暂缓 | 中高 | 很低 | 需要实验体系或历史实验记录 |

## 2. 为什么这样筛

### 优先做 A

- 报告里最直接的监督信号来自 DoLPHiN，且条目规模已经足够做第一版 benchmark。
- 这是一个“重要且未被充分解决”的问题。现有 SweetNet、glyBERT、GlycanML 多数仍偏 glycan，不是真正的长链 polysaccharide 问题。
- 不需要先建实验平台，单卡 GPU 就能做出第一版结果。
- 最容易形成论文、数据子集、评测脚本三件套。

### 同步做 D，但只做 A 所需的最小闭环

- 当前真正的瓶颈不是模型，而是跨库对齐、证据追踪、表示标准不一致。
- 但不建议一开始做“大而全 KG”。先做能支撑 A 的最小 KG：`结构 -> 来源 -> 功能标签 -> 证据 DOI -> 数据来源`。
- 这样既能解决 ETL 和可追溯性问题，又不会把工程范围做炸。

### F 作为第二阶段，而不是起手式

- 预训练有研究价值，但前提是表示、ETL、下游任务先稳定。
- 如果直接从 F 开始，会陷入“大模型先行、任务定义不清”的常见陷阱。
- 正确顺序是：先用 A 证明“现有表示不够”，再用 F 去补标签稀缺与泛化不足。

### 暂缓 B/C/E

- B 最大问题不是模型，而是高质量“谱图-结构”对照集不足，且跨仪器域偏移严重。
- C 依赖可靠的性质预测器、约束规则和可验证候选，当前做会变成“生成一堆不可用结构”。
- E 需要实验闭环或历史实验记录，当前仓库和报告都没有给出现成抓手。

## 3. 建议的总体研究组合

建议采用 `A 主线 + D 底座 + F 预研` 的组合。

### 主问题

构建一个面向多糖的结构-功能预测基准与模型体系，重点回答两个问题：

1. 现有 glycan 模型迁移到 polysaccharide 任务时，在哪些结构特征上失效？
2. 显式加入重复单元、分支、修饰、分子量和证据权重后，能否提升跨来源泛化？

### 第一篇论文的最现实落点

- 题目方向：`Polysaccharide Structure-Function Benchmark and Evidence-Aware Modeling`
- 核心贡献：
- 一个可复现的多糖结构-功能数据子集
- 一套多糖表示与评测协议
- 一个证据加权的 baseline/model family
- 一份可解释的构效分析

## 4. 12 周可执行计划

### 第 1-2 周：把数据问题定义清楚

- 建立仓库结构：`data_raw`、`data_interim`、`data_processed`、`src/etl`、`src/models`、`src/kg`、`docs`
- 定义统一 schema：
- `poly_id`
- `source_db`
- `raw_representation`
- `canonical_representation`
- `monomer_composition`
- `linkage`
- `branching`
- `modification`
- `mw_or_range`
- `organism_source`
- `function_label`
- `evidence_type`
- `doi`
- `license`
- 先手工抽样 100 条报告里重点提到的 DoLPHiN/CSDB 条目，验证字段能否稳定落表
- 选定训练主表示，建议优先 `canonical IUPAC`，并保留 `WURCS` 作为兼容字段

阶段出口条件：

- 100 条样本中，90% 以上能落到统一 schema
- 明确哪些字段是强监督，哪些字段只能做弱标签或元数据

### 第 3-4 周：完成 ETL v0 和数据子集 v0

- 抓取 DoLPHiN 为主，CSDB 只补结构/NMR/来源信息，不追求全量同步
- 建立 GlyTouCan 映射表，作为跨库锚点
- 对功能标签做证据分级：
- `in_vitro`
- `animal`
- `clinical`
- 构建三类数据拆分：
- 随机拆分
- 按来源数据库拆分
- 按物种或属级拆分
- 输出 `dataset_v0`、`schema.md`、`data_card.md`

阶段出口条件：

- 有一个可训练的数据子集
- 每条记录都带来源、证据、版本和许可字段
- 数据处理脚本能从原始数据一键重建派生数据

### 第 5-6 周：先做最硬的 baseline

- Baseline 1：手工特征 + XGBoost/LightGBM
- Baseline 2：序列表示 + Transformer
- Baseline 3：图表示 + GNN
- 先不追求 fancy 架构，先回答“多糖任务到底有没有信号”
- 训练时加入证据权重和标签噪声处理
- 固定评测指标：
- 分类：AUROC、AUPRC、macro-F1
- 回归：MAE、RMSE
- 泛化：leave-one-source-out、leave-one-genus-out

阶段出口条件：

- 三套 baseline 都能稳定复现
- 至少确认一个任务存在可学习信号，不是纯噪声

### 第 7-8 周：做可解释性和误差分解

- 提取 motif、链接、修饰、分子量等特征的重要性
- 对错误样本做分类：
- 表示不完整
- 标签冲突
- 物种偏移
- 实验体系偏移
- 做第一版构效规则总结
- 同期产出最小 KG v0，只保留支撑当前任务的节点和边

阶段出口条件：

- 能说清楚模型为什么对、为什么错
- KG v0 可以追溯任意预测样本对应的证据来源

### 第 9-10 周：决定是否进入 F 预训练

- 如果 A 的监督模型已经有稳定提升：
- 开始做 F 的轻量版预训练，用 GlyTouCan 结构语料做表示初始化
- 如果 A 的提升不稳定：
- 不做预训练，优先返工标签质量和表示标准
- 预训练只做一个最小问题：
- 图对比学习初始化 GNN
- 或 masked sequence modeling 初始化 Transformer

阶段出口条件：

- 明确预训练是否真的带来少样本或跨来源泛化增益

### 第 11-12 周：收敛成一篇能投的资源+方法稿

- 固化数据版本、ETL 版本和 split 版本
- 产出论文骨架：
- 问题定义
- 数据构建
- 表示与模型
- 评测协议
- 结果与构效解释
- 产出开源内容：
- `dataset_v0`
- `benchmark scripts`
- `KG v0`
- `model checkpoints`
- `reproduce.md`

阶段出口条件：

- 至少形成一篇资源/方法混合型稿件的完整初稿
- 任何一个结果都可以通过脚本复现

## 5. 现在不该做什么

- 不要先做完整 18 个月大图谱平台。
- 不要先做生成模型。
- 不要先做“基础模型”大规模预训练。
- 不要把谱图任务作为第一篇文章。
- 不要在数据 schema 还不稳定时频繁换表示标准。

## 6. 风险与对应止损点

### 风险 1：DoLPHiN 字段比预期更难结构化

止损动作：

- 缩小问题，只保留 2-3 个功能标签最清晰的任务
- 先发数据规范和 benchmark，不强推复杂模型

### 风险 2：标签过噪，模型无法学到稳定信号

止损动作：

- 改成证据分层建模
- 做弱监督/多实例学习
- 先聚焦单一应用场景，例如免疫调节或抗氧化

### 风险 3：跨库映射成本过高

止损动作：

- 第一阶段只做 DoLPHiN 主数据集
- CSDB/GlyTouCan 只作为补充字段，不强求全覆盖

### 风险 4：预训练没有带来提升

止损动作：

- 把 F 降级为附录实验
- 主稿仍以 A + D 为主

## 7. 建议的里程碑与交付物

| 时间 | 交付物 | 最低合格标准 |
| :--- | :--- | :--- |
| 第 2 周 | schema v0 + 手工标注样本 | 样本字段可稳定解析 |
| 第 4 周 | dataset v0 + ETL 脚本 | 可以一键重建数据 |
| 第 6 周 | baseline 结果表 | 至少一个任务存在可学习信号 |
| 第 8 周 | KG v0 + 误差分析 | 预测结果可追溯到证据 |
| 第 10 周 | 预训练是否继续的决策 | 有明确 go/no-go 结论 |
| 第 12 周 | 论文初稿 + 开源包 v0 | 外部可复现 |

## 8. 一句话版本

最值得马上做的，不是“多糖大模型”也不是“生成新多糖”，而是先把 `DoLPHiN 驱动的结构-功能预测` 做成一个可复现 benchmark，并用 `最小知识图谱` 把数据对齐、证据追踪和可解释性补齐；等这条主线跑通后，再决定是否上 `自监督预训练`。
