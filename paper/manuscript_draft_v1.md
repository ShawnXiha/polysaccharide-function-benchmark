# Ontology-Aware Tail-Sensitive Function Retrieval on a DoLPHiN-Derived Polysaccharide Knowledge Graph

## Abstract

Polysaccharide bioactivity studies contain rich but weakly structured evidence linking polymer composition, source organisms, functional annotations, disease relevance, and literature provenance. However, these signals are rarely organized as graph-native resources that support retrieval-oriented inference or ontology-aware reasoning. In this work, we convert DoLPHiN into a heterogeneous polysaccharide knowledge graph with 5,078 polysaccharide nodes linked to organism, monosaccharide, glycosidic bond, function, disease, and publication entities. We then study masked `poly-function` link prediction as a controlled benchmark for graph-enhanced function retrieval. Our experiments show that, under a clean setting without disease side information, an interpretable meta-path retrieval baseline is substantially stronger than the current hetero GNN and hybrid GNN variants on this graph. Disease-aware signals provide a strong upper bound, but they are better interpreted as auxiliary semantics than as the main causal source of structure-function inference. To improve rare-label behavior, we introduce a confidence-gated parent/child ontology propagation scheme that injects ontology evidence directly into the retrieval stage. This variant does not become a new overall best model, but it yields a stable tail-sensitive gain: across 16 paired seeds and 16,000 masked edges, tail micro filtered Hits@3 improves from 0.0552 to 0.1021, with a one-sided paired McNemar p-value of 0.03125 and a tail filtered MRR gain of 0.0395 with permutation p-value 0.0002499. These results suggest that the current value of the DoLPHiN-derived knowledge graph lies first in interpretable relation-structured retrieval, while ontology support becomes useful when it is expressed as selective parent/child propagation rather than coarse family smoothing.

## 1 Introduction

Long-chain polysaccharides occupy a difficult but important space in biomolecular research because their biological roles emerge from heterogeneous structural cues, source-dependent context, and sparse functional evidence. Unlike many small-molecule or protein prediction settings, polysaccharide studies are fragmented across domain-specific databases and literature records, and their evidence is often stored as semi-structured text rather than machine-actionable graphs. This fragmentation limits not only downstream prediction, but also mechanistic querying, provenance tracking, and hypothesis generation.

DoLPHiN is a useful starting point because it aggregates polysaccharide-centered records that already expose several relevant evidence channels, including source organisms, monomer composition, linkage patterns, functional labels, disease associations, and DOIs. Yet the raw records are not immediately suitable for graph learning or structured retrieval. Important entities still appear as free text, heterogeneous relations are not explicitly typed, and provenance is not arranged in a way that supports transparent graph-native evaluation.

This paper addresses that gap by transforming DoLPHiN into a heterogeneous polysaccharide knowledge graph and then using this graph to study function retrieval under masked `poly-function` prediction. Our goal is deliberately narrower than claiming a universally optimal predictor. Instead, we ask three concrete questions. First, can a DoLPHiN-derived knowledge graph provide usable supervision for function retrieval at all? Second, on the current graph version, are the useful signals better captured by interpretable relation-derived retrieval features or by hetero GNN message passing? Third, can ontology support improve the rare-label regime without materially distorting the stronger global baselines? Figure 1 summarizes the overall data-to-graph-to-retrieval workflow that frames these questions.

Our empirical answers are specific. The knowledge graph is already useful: a clean meta-path retrieval baseline reaches a test macro-F1 of 0.3046 in multi-label function prediction, showing that source, monomer, bond, and publication relations carry real predictive signal even without disease information. At the same time, the current hetero GNN and hybrid GNN variants remain much weaker than explicit retrieval on this graph, which suggests that the main bottleneck is not the absence of graph structure, but the mismatch between the current graph representation and what end-to-end message passing can learn from it. Disease-aware variants are substantially stronger, but they should be interpreted as upper-bound side information because disease nodes are tightly coupled to the target functions. The overall benchmark comparison is summarized in Figure 2 and Table 2.

The final part of the paper focuses on the long tail. Many ontology formulations we tried either acted as near no-ops or degraded overall ranking by oversmoothing function families. The useful variant emerged only after converting ontology support into confidence-gated parent/child propagation that selectively reinforces low-support function candidates during retrieval. This mechanism preserves the overall disease-aware upper bound while consistently improving tail-sensitive behavior. The main contribution of the paper is therefore not a generic claim that ontology always helps, but a more specific conclusion: on the current DoLPHiN-derived graph, ontology becomes useful only when expressed as selective graph-native evidence for tail-sensitive retrieval.

## 2 Methods

We build a heterogeneous knowledge graph centered on `Polysaccharide` entities extracted from DoLPHiN. Each polysaccharide node is linked to normalized `Organism`, `Monosaccharide`, `GlycosidicBond`, `Function`, `Disease`, and `Publication` nodes. The current graph contains 5,078 polysaccharides, 1,487 organism nodes, 23 monosaccharide nodes, 323 glycosidic bond nodes, 66 function nodes, 14 disease nodes, and 2,386 publication nodes. The resulting graph preserves both typed structure and record-level provenance, which allows the same resource to support querying, graph export, and controlled predictive evaluation. The graph statistics are summarized in Table 1.

Our primary task is masked `poly-function` link prediction. For each evaluation instance, one known `Polysaccharide`-to-`Function` edge is hidden, and the model ranks all candidate function nodes for that polysaccharide. We report filtered ranking so that other true functions already attached to the same polysaccharide are removed from the candidate set before rank computation. This is essential because the task is intrinsically multi-label. We further stratify evaluation by label support into tail, mid, and head groups, since aggregate metrics are otherwise dominated by common functions.

The strongest clean baseline is a meta-path retrieval model that constructs explicit feature representations from graph relations and applies local k-nearest-neighbor label propagation. In practice, this approach is more effective than current hetero GNN variants on our graph because it directly exposes relation-derived evidence rather than relying on weakly featured non-polysaccharide nodes to acquire discriminative embeddings through message passing. We also evaluate disease-aware variants as upper-bound settings, where disease-conditioned compatibility is injected into the retrieval stage and then mildly frequency-adjusted to reduce the bias toward frequent labels.

To target the long tail, we introduce ontology-aware parent/child propagation. A manually curated function hierarchy is represented as directed parent/child relations rather than undifferentiated family membership. During retrieval, the base disease-aware score first forms a shortlist of plausible function labels. Ontology propagation is then activated only when the local evidence is sufficiently uncertain and when there is enough seed support in the relevant parent region. The propagated contribution is further modulated by an adaptive confidence gate, preventing coarse family smoothing from overwhelming the original neighborhood evidence. This design is motivated by our earlier failures with hierarchy bonuses and family-level smoothing, which either had negligible effect or degraded overall ranking. Figure 1 highlights where ontology enters the pipeline: not as a post-hoc family bonus, but as selective evidence propagation inside the retrieval stage.

## 3 Experiments

We first evaluate whether the graph is useful for function prediction at all. In the clean setting, Meta-Path reaches a test macro-F1 of 0.3046 and a test exact match of 0.4144, whereas clean Hetero GNN and clean Hybrid Hetero GNN remain far below this level. This result establishes that the current DoLPHiN-derived graph already contains usable signal, but the signal is most accessible through explicit relation-derived retrieval rather than through the current end-to-end hetero GNN stack. When disease side information is added, Meta-Path improves further to a test macro-F1 of 0.4560, confirming that disease carries strong auxiliary semantics. We therefore use disease-aware retrieval as an upper-bound reference rather than the clean main claim. The main predictive results are summarized in Table 2, while Figure 2A visually contrasts the clean and disease-aware function-prediction settings.

We then turn to masked `poly-function` link prediction, where the main disease-aware baseline is the disease-conditioned vote combined with a frequency-adjusted disease prior. This baseline is strong in aggregate ranking, but support-stratified analysis reveals a persistent long-tail weakness. A large set of long-tail interventions was tested, including rare-label neighbor expansion, source-constrained reranking, structure-aware candidate generation, motif-based signatures, and several ontology variants. Most either failed to change the shortlist or traded away head performance without improving the tail. This negative evidence is important because it narrows the plausible mechanisms: the tail problem is not solved by stronger family smoothing or by generic structural bonuses. Figure 2B and Figure 2C focus on the specific comparison that survives this screening process.

The useful ontology mechanism is the confidence-gated parent/child propagation model. On the main tuned split, it matches the disease-aware baseline on filtered Hits@3 while improving tail micro filtered Hits@3 from 0.1667 to 0.3333. To determine whether this effect is stable rather than accidental, we ran a paired significance protocol over 16 seeds, yielding 16,000 masked edges with edge-level records for both the baseline and the ontology variant. Under this protocol, the ontology model leaves the global disease-aware profile essentially unchanged, with filtered Hits@3 moving from 0.9088 to 0.9085 and filtered Hits@5 staying at 0.9366. However, the tail-sensitive gains remain consistent: tail micro filtered Hits@3 increases from 0.0552 to 0.1021, the directional one-sided McNemar test gives p = 0.03125, and the tail filtered MRR gain is 0.0395 with permutation p = 0.0002499. No seed shows a tail regression. Figure 3 presents the seed-level non-regression pattern and the paired significance summary, while Table 3 lists the corresponding numerical results.

These results support a narrow but defensible conclusion. The ontology mechanism is not a new overall best scorer, and it slightly lowers overall filtered MRR. Nevertheless, it produces a stable tail-sensitive improvement under the exact setting where ontology is expected to matter most. This makes it appropriate as an ontology-enhanced supplement to the main disease-aware upper-bound baseline rather than as a replacement for the core retrieval pipeline.

## 4 Discussion

The results indicate that the immediate value of the DoLPHiN-derived knowledge graph lies in structured retrieval rather than in the current generation of graph neural architectures. This does not imply that GNNs are unsuitable for polysaccharide graphs in principle. Instead, it suggests that the present graph version exposes stronger signal through explicit relation structure than through node feature learning. Non-polysaccharide nodes remain semantically weak, graph connectivity is sparse relative to the prediction difficulty, and several critical structural regularities are more naturally expressed as typed evidence than as messages propagated through shallow neighborhoods.

The disease-aware results should also be interpreted carefully. They demonstrate that disease information is strongly predictive, but this is precisely why it should not define the clean main claim. If disease is tightly coupled to the target function labels, then disease-aware retrieval is better understood as an upper-bound semantic enhancement. Our cleaner scientific claim is therefore that the knowledge graph already supports useful function retrieval without disease, while disease and ontology help explain where additional side information can be injected and how that affects the long tail.

The ontology findings are especially instructive because they constrain future work. Coarse family bonuses and undirected hierarchy smoothing were ineffective. Parent/child propagation became useful only when it was selective, confidence-gated, and integrated directly into retrieval rather than appended as a generic reranking bonus. This pattern suggests that ontology in this domain is most valuable as a targeted evidence-routing mechanism for under-supported labels, not as a broad semantic regularizer. A practical next step is therefore not to enlarge the ontology blindly, but to refine local parent/child evidence and combine it with richer structural signatures that are currently absent from the graph.

## 5 Related Work

Existing work relevant to this study falls into three partially overlapping groups. The first group concerns polysaccharide databases and glycan resources. These resources are indispensable for curation and lookup, but they are usually optimized for record access rather than for graph-native retrieval benchmarks. Our work differs in emphasis: the objective is not only to store polysaccharide records, but to normalize them into a typed evidence graph that can support controlled predictive tasks with provenance preserved.

The second group covers knowledge-graph-based biological prediction. In many biomedical settings, graph learning has been used to connect heterogeneous entities such as molecules, diseases, genes, phenotypes, and publications. The dominant intuition in that literature is that richer heterogeneous connectivity should favor graph neural models or embedding-based reasoning. Our results only partly align with that expectation. On the current DoLPHiN-derived graph, explicit relation-derived retrieval remains stronger than the hetero GNN baselines, which suggests that graph-native biological prediction should not automatically be equated with end-to-end message passing. In sparse and weakly featured scientific graphs, interpretable retrieval may still be the more faithful first-line model.

The third group concerns ontology-aware reasoning and long-tail prediction. Ontologies are often used as hierarchical regularizers, label smoothers, or post-hoc propagation tools. Our experiments show that these generic formulations are not sufficient in the present setting. Family-level bonuses and coarse hierarchy smoothing were ineffective or harmful. The effective ontology formulation is narrower: parent/child relations must be injected as selective evidence propagation under confidence control. This places our work closer to targeted tail-sensitive calibration than to generic ontology regularization.

## 6 Limitations

This paper should be read within a deliberately bounded scope. First, the clean and disease-aware settings answer different questions. The clean setting tests whether the graph itself contains useful structure-function signal, whereas the disease-aware setting estimates a practical upper bound when auxiliary semantics are available. Because disease is tightly coupled to the target functions, the disease-aware results should not be interpreted as a pure structure-only mechanism.

Second, the current function hierarchy is intentionally modest. It is sufficient to test whether ontology can help the tail, but it is not a comprehensive biological ontology of polysaccharide function. The parent/child propagation result should therefore be interpreted as proof of mechanism, not as the final possible ontology result.

Third, the graph remains structurally incomplete in several ways. Richer motif-level representations, external glycan ontologies, and more expressive provenance-aware evidence typing are not yet integrated. These missing components may matter for both cleaner biological interpretation and stronger end-to-end graph learning. The present manuscript therefore supports a constrained claim: the current graph is already useful for interpretable retrieval, and selective ontology propagation provides a stable tail-sensitive gain on top of that retrieval stack.

## 7 Conclusion

We presented a DoLPHiN-derived heterogeneous polysaccharide knowledge graph and used it to study graph-enhanced function retrieval. The main empirical finding is that explicit meta-path retrieval remains the strongest clean approach on the current graph, while disease-aware retrieval defines a strong upper-bound regime. Within that upper-bound setting, confidence-gated parent/child ontology propagation yields a reproducible tail-sensitive improvement without changing the overall ranking profile in a meaningful way. Taken together, these results position the current graph as a useful substrate for interpretable retrieval and targeted ontology-aware reasoning, while also clarifying what kinds of graph signal remain underexploited by current hetero GNN baselines.
